# 固定日志文件设计

## 目标

将上位机日志记录流程改为固定输出到项目根目录的 `record.csv`：
- 点击“开始记录”时立即创建或覆盖 `record.csv`
- 写入 CSV 表头
- 记录期间每条遥测数据直接追加到文件
- 点击“停止记录”时直接完成保存，不再弹出导出对话框

## 设计

### 日志服务

`app/services/logger.py` 负责管理记录状态和 CSV 文件句柄：
- `start(path)` 打开固定路径文件并写入表头
- `add(data)` 在保留内存记录的同时向文件追加一行并刷新
- `stop()` 关闭文件，确保数据落盘

### 界面行为

`app/ui/main_window.py` 负责固定路径与交互：
- 将记录按钮文案改为“停止记录”
- 使用项目根目录 `record.csv` 作为唯一输出文件
- 开始记录时提示目标路径
- 停止记录时提示保存完成

## 风险与处理

- 重复开始记录会覆盖旧文件：与“开始一次新记录”的语义一致
- 异常退出可能导致末尾少量数据未写入：每次追加后立即刷新以降低风险
