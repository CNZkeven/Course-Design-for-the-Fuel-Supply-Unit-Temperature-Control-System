### 1.2 下位机程序设计流程

下位机程序位于 `code/USER/main.c`，主函数 `main()` 执行流程如下：

1. **初始化阶段**：
   - 设置 NVIC 中断优先级分组为 Group_2
   - 初始化系统延时（168MHz 主频）
   - 初始化 USART1 串口（115200, 8N1）
   - 初始化 LED（心跳指示）
   - 初始化 LCD（状态与参数显示）
   - 初始化 ADC1（PA6 模拟输入）
   - 初始化 DAC1（PA4 电压输出）
   - 绘制 LCD 初始界面

2. **PID 初始默认参数**：
   - P = 50.0, I = 0.1, D = 0.0, limit = 100.0
   - 默认目标温度 set_temp = 60.0℃
   - 超温阈值 over_limit = 75.0℃

3. **主循环（约 100ms 一个周期）**：
   ```
   while(1) {
       ① ADC 采样 20 次平均 → adcx
       ② adc2temp(adcx) → temp（当前温度）
       ③ 超温判断：if temp >= over_limit → 报警停机
       ④ 刷新 LCD 显示
       ⑤ if 运行中 → pid_realize() → control_to_dac_code() → dac_output()
          else → 输出零点（DAC=2048, 0V）
       ⑥ LED 翻转（心跳）
       ⑦ delay_ms(100)
       ⑧ printf() 上报 CSV 遥测帧
       ⑨ usart_process() 解析上位机命令
   }
   ```

### 1.3 上位机程序设计流程

上位机程序位于 `app/` 目录，基于 PySide6 + pyqtgraph 实现：

1. **启动流程**：`app/main.py` → 创建 `MainWindow` → 加载本地保存的参数 → 显示界面
2. **连接串口**：选择端口和波特率 → 调用 `SerialManager.connect_port()` → QSerialPort 打开
3. **参数下发**：输入参数 → 点击"下发到下位机" → `send_batch()` 分时发送（间隔120ms）
4. **数据接收**：`readyRead` 信号 → `_on_ready_read()` 缓冲分帧 → `_on_line()` 解析遥测
5. **曲线绘制**：每帧数据追加到缓存数组 → `pyqtgraph` 刷新温度/设定/输出三条曲线
6. **数据记录**：点击"开始记录" → `Logger.start()` 创建 CSV → 每帧写入 → 停止后 flush 关闭

## 三、实验文字说明

### 3.1 实验目的

设计一个以 STM32F407 为控制核心的燃油供应单元温度控制系统，使用 PT100 温度传感器构成温度检测闭环，通过 PID 算法控制可控硅调压器，将被控对象温度稳定在设定值，同时通过 Python 上位机实现参数整定、曲线显示和数据记录。

### 3.2 实验设备与条件

- STM32F407ZGT6 开发板
- PT100 温度传感器及信号调理电路
- 可控硅电子调压器（电压输入控制方式）
- USB 转串口模块
- 加热对象（模拟燃油供应单元加热装置）
- PC 电脑（运行 Python 上位机）

### 3.3 实验步骤

1. 将 STM32 固件编译并烧录至开发板（使用 CMake + arm-gcc 交叉编译，J-Link 下载）。
2. 连接硬件：PT100 信号调理输出接 PA6，DAC 输出 PA4 接可控硅控制端，串口接 USB 转串口模块。
3. 上电后观察 LCD 显示，确认温度读数、设定值、PID 参数正常（LCD 界面分为 PROCESS、SIGNAL、CONTROL 三个面板，顶部状态栏显示 READY/RUN/ALARM 状态）。
4. 在 PC 端运行 `python app/main.py` 启动上位机。
5. 选择串口端口，设置波特率 115200，点击"连接"。
6. 在参数设置区输入设定温度（如 65℃）、P（如 12.0）、I（如 0.1）、D（如 18.0）、超温阈值（如 75℃），点击"下发到下位机"。
7. 点击"启动"按钮，系统进入闭环温控状态。
8. 观察上位机温度曲线，待温度趋近设定值后，可尝试调整 PID 参数观察超调量和稳定速度的变化。
9. 点击"开始记录"，系统将实验数据实时保存到 `record.csv`。
10. 若需测试超温保护，可降低超温阈值验证报警停机功能。
11. 实验结束后点击"停止记录"和"停止"按钮。

### 3.4 实验安全注意事项

- 系统具备超温保护功能：当温度 ≥ 超温阈值时，自动置报警并关断加热输出
- 启动前会先检测当前温度是否已超温，若已超温则拒绝启动
- 超温保护优先级高于正常控温逻辑

## 六、PID 控制算法

### 6.1 算法类型

本系统采用**位置式 PID 控制算法**。

### 6.2 数学表达式

```
e(k) = 目标温度 - 当前温度

积分项 = 积分项 + e(k) × dt

微分项 = [e(k) - e(k-1)] / dt

u(k) = P × e(k) + I × 积分项 + D × 微分项
```

其中 dt = 0.1s（100ms 控制周期）。

### 6.3 程序实现（来自 main.c）

```c
float pid_realize(Error *error, PID *pid, float current, float target) {
    float error_value = target - current;              // e(k) = 目标温度 - 当前温度
    float d_error;
    float output;

    error->Current_Error += error_value * dt;          // 对误差积分
    d_error = (error_value - error->Last_Error) / dt;  // 一阶差分近似微分

    output = pid->P * error_value +                    // 比例项
             pid->I * error->Current_Error +           // 积分项
             pid->D * d_error;                         // 微分项

    // 输出限幅
    if (output > pid->limit) output = pid->limit;
    if (output < -pid->limit) output = -pid->limit;

    error->Last_Error = error_value;                   // 保存上一次误差
    return output;
}
```

### 6.4 PID 三项作用说明

| PID 参数 | 符号 | 作用 |
| --- | --- | --- |
| 比例系数 P | Kp | 按当前误差大小成比例输出控制量，误差越大输出越强；P 越大响应越快但易引起振荡 |
| 积分系数 I | Ki | 累积历史误差，消除稳态误差；I 过大会导致积分饱和和超调加剧 |
| 微分系数 D | Kd | 根据误差变化率提前反应，抑制温度变化过快导致的超调；D 过大会放大噪声 |

### 6.5 工程化处理措施

1. **输出限幅**：PID 输出被限制在 [-limit, +limit] 范围内，防止控制量超出执行机构允许范围
2. **负输出钳位**：在 `control_to_dac_code()` 中，负控制量被钳位为 0，因为执行端是加热装置，只需单向输出
3. **在线参数整定**：PID 参数可通过上位机串口命令实时修改，无需重新编译固件

### 6.6 相关数据结构

```c
typedef struct {
    float P, I, D, limit;  // PID 三个系数 + 输出限幅
} PID;

typedef struct {
    float Current_Error;    // 积分累加量
    float Last_Error;       // 上一次误差（用于微分项）
    float Previous_Error;   // 预留给增量式 PID 扩展
} Error;
```

## 七、控制策略

### 7.1 总体控制策略

系统采用**典型的闭环温度控制结构**，控制策略要点如下：

1. **采集**：每个控制周期对 ADC 通道 6 做 20 次采样平均，降低噪声影响
2. **换算**：通过标定后的线性映射公式将 ADC 值转换为 0~150℃ 温度值
3. **控制**：位置式 PID 算法计算控制输出
4. **执行**：控制量线性映射为 DAC 码值（2048~4095），驱动可控硅调压器
5. **保护**：超温检测优先于控制逻辑，超温立即报警停机
6. **通信**：串口不直接参与 PID 运算，仅承担参数整定和状态监视任务

### 7.2 温度采集策略

**传感器标定**：
- ADC 满量程 12 位（0~4095）
- 外部调理电路将 PT100 信号映射为 0~5V
- ADC 侧零点标定：0 code → 0℃
- ADC 满量程：4095 code → 150℃
- 温度换算使用完整量程 [0, 4095]

**采样滤波**：
- 每轮采集 20 次，每次间隔 5ms
- 取平均值，降低噪声抖动
- 仅 ADC 采样阶段就约消耗 100ms

**换算公式**：
```
temperature = max(0, min(adc_value, 4095)) × 150 / 4095
```

**对应实现说明（2026-03-20）**：`code/USER/temp_conversion.c` 已同步为 `ADC_ZERO_CODE=0`、`ADC_FULL_SCALE_CODE=4095`，即 0V 对应 ADC=0，5V 对应 ADC=4095。

### 7.3 控制输出策略

**DAC 输出映射**：
- DAC 零点码值：2048（对应 0V 输出）
- DAC 满量程码值：4095（对应 5V 输出）
- PID 控制量 [0, limit] 线性映射到 DAC [2048, 4095]
- 负控制量钳位为 0（仅加热，单极控制）

**映射公式**：
```
dac_code = 2048 + (control_output × (4095 - 2048)) / output_limit
```

**电压输出**：
- DAC 码值先转为毫伏值写入 DAC 寄存器：`dac_mv = dac × 3300 / 4096`
- 再通过独立标定函数换算为实际控制电压
- 实际控制电压 = (dac_code - 2048) × 5.0 / (4095 - 2048)

### 7.4 超温保护策略

```
优先级：超温保护 > 正常控温 > 串口通信

if (当前温度 >= 超温阈值) {
    → 置报警标志 alarm = 1
    → 停止控温 run_enabled = 0
    → DAC 输出零点（停止加热）
}

启动前检查：
if (当前温度 >= 超温阈值) {
    → 拒绝启动
    → 提示 "Over limit, cannot start"
}
```

### 7.5 串口通信策略

**下行（MCU → PC）**：每个控制周期上报一帧 CSV 遥测数据

```
格式：TEMP,SET,P,I,D,DAC,VOLT,RUN,ALARM,LIMIT
示例：45.21,60.00,50.00,0.10,0.00,2140,1.233,1,0,75.00
```

**上行（PC → MCU）**：文本命令，以 `\r\n` 结尾

| 命令 | 格式 | 功能 |
| --- | --- | --- |
| START | `START\r\n` | 启动加热 |
| STOP | `STOP\r\n` | 停止加热 |
| TEMP | `TEMP:xx.xx\r\n` | 设置目标温度 |
| P | `P:xx.xx\r\n` | 设置比例系数 |
| I | `I:xx.xx\r\n` | 设置积分系数 |
| D | `D:xx.xx\r\n` | 设置微分系数 |
| LIMIT | `LIMIT:xx.xx\r\n` | 设置超温阈值 |
| GET | `GET\r\n` | 查询当前参数 |

**接收实现**：中断收字节 + 缓冲区 + `\r\n` 分帧 + 主循环解析，不在中断中执行复杂操作。

### 8.4 PID 参数对控制效果的影响分析

本次实验使用参数 P=12.0, I=0.1, D=18.0：

- **P=12.0（比例项）**：适中的比例系数，使系统在较大温差时有足够的控制力度（误差 13℃ 时输出 = 12×13 = 156，被限幅到 100），同时避免了过大的震荡。
- **I=0.1（积分项）**：较小的积分系数，缓慢积累以消除稳态误差，但积累速度慢，避免积分饱和导致的大幅超调。
- **D=18.0（微分项）**：较大的微分系数，在温度快速接近设定值时产生反向制动力，有效抑制了超调。从实验数据看，首次超调仅 1.39℃，说明微分项起到了关键的抑超调作用。

### 8.5 DAC 输出特征分析

从实验数据观察：
- 升温阶段：DAC 持续输出 4094（满量程，4.998V），说明温差大时 PID 输出超过限幅，系统以最大功率加热
- 达到设定值后：DAC 在 2048（0V）和 4094（4.998V）之间频繁切换，呈"开关式"振荡特征
- 这种现象是因为微分项 D=18.0 较大，当温度略超设定值时微分制动很强，输出直接降到 0；温度稍有回落又恢复加热

### 8.6 进一步优化建议（供报告撰写参考）

1. 可适当降低 D 值（如 D=10），减少控制输出的频繁切换
2. 可适当增大 I 值（如 I=0.2），加快消除稳态误差
3. 可引入积分分离策略：误差大时关闭积分项，减少积分饱和
4. 可引入微分先行策略：对测量值而非误差求微分，避免设定值跳变时微分项冲击
5. 可记录不同 PID 参数组合下的曲线进行对比分析
